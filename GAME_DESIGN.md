## 1. 游戏概述

### 1.1 游戏类型
- **类型**: 生存冒险 / Roguelike 风格
- **平台**: Python + Pygame (桌面端)
- **分辨率**: 1400 × 900 px
- **目标**: 在荒岛生存 30 天获救

### 1.2 核心循环
```
捕捞 → 探索 → 战斗/合成 → 建造 → 推进天数 → 获救
```

### 1.3 初始资源
| 物品 | 数量 |
|------|------|
| 木材 | 15 |
| 石头 | 8 |
| 鱼 | 3 |
| 绳索 | 3 |
| 木棍 | ×1 (武器, 耐久30) |

## 2. 核心系统

### 2.1 基础行动 (行动点: 3/天)

| 行动 | 消耗 | 效果 |
|------|------|------|
| 🎣 捕捞 | 1pt | 获得 1-4 条鱼，消耗武器耐久1点 |
| 🗺 探索 | 1pt | 随机获得资源，可能遭遇敌人，消耗武器耐久1点 |
| 🍖 进食 | 0pt | 消耗1条鱼，恢复饱腹度40 |
| 💊 治疗 | 0pt | 消耗1个神秘果实，恢复生命30 |
| 🔨 合成 | 0pt | 消耗材料制作装备/工具 |
| 🏗 建造 | 0pt | 消耗材料建造建筑 |

### 2.2 每日结算
- 饱腹度 -12/天
- 饱腹度为 0 时生命 -15/天
- 30% 概率触发灾害 (瞭望塔降低至 15%)
- 每日自动尝试生成任务
- 天气随机变化

### 2.3 天气系统
| 天气 | 触发概率 | 视觉效果 |
|------|----------|----------|
| ☀️ 晴天 sunny | 70% | 正常 |
| ☁️ 多云 cloudy | 30% | 云层变厚 |
| 🌧️ 雨天 rainy | 30% | 雨滴粒子效果 |
| ⛈️ 暴风雨 stormy | 30% | 暴雨+龙卷风粒子 |
| 🌫️ 雾天 foggy | 30% | 半透明灰雾覆盖 |

> 每天 `_next_day()` 有 30% 概率切换天气状态

### 2.4 灾害系统
灾害类型 (随机, 瞭望塔减半):
| 灾害 | 基础伤害 | 防御墙减免 |
|------|----------|------------|
| 暴风雨 | 15 | 40% |
| 海啸 | 25 | 40% |
| 龙卷风 | 20 | 40% |
| 瘟疫 | 18 | 40% |

石屋额外减免 20% 伤害。

## 4. 装备与耐久度

### 4.1 耐久度机制
- **武器**: 捕捞、探索、战斗攻击各消耗 1-2 点耐久
- **护甲**: 受到灾害伤害时消耗 2 点耐久
- **木棍特殊**: 耐久耗尽时自动重置为 30 (不会真正损坏)
- **其他武器**: 耐久耗尽时变为无武器状态，物品保留在背包

### 4.2 自动装备
- 制作武器后自动装备 (替换当前武器)
- 制作护甲后自动穿戴
- 任务奖励武器时自动装备

### 4.3 UI 显示
- 状态栏: `⚔ 石矛: 47` (绿色>15, 橙色≤15, 红色≤5)
- 动作栏: 武器/护甲快捷显示
- 背包: 物品旁显示耐久度数值

## 5. 剧情与任务

### 5.1 开场动画 (8页)
游戏启动时展示 8 页渐进式叙事，讲述主角醒来后逐步找回记忆的故事。打字机效果，每页 6 秒，可按任意键跳过。

故事线: 醒来 → 搭建庇护所 → 发现照片 → 20天后 → 暴风雨发现航海日志 → 日志残页 → 回忆起家人(林雨晴+小海) → 最终获救

### 5.2 漂流瓶故事系统
- **触发**: 第2天起，每天 `_next_day()` 有 8% 概率发现
- **内容**: 8 页独立故事碎片
- **追踪**: `Player.pages_found` 记录已解锁页码
- **阅读**: 背包界面点击【故事】按钮打开阅读面板

### 5.3 任务系统 (Quest)
**触发**: 每天 `_next_day()` 有 35% 概率生成新任务

| 类型 | 示例 | 条件 |
|------|------|------|
| collect | 收集5条鱼 | `p.has(target) >= target_count` |
| build | 建造小木屋 | `any(b.name == target for b in buildings)` |
| combat | 击败3个敌人 | `p.enemy_kills >= target_count` |
| survive | 再存活N天 | `p.day >= target_count` |

**奖励**: 经验、生命、金属、神秘果实、武器

### 5.4 结局系统 (5种)
| 结局 | 条件 | 主题 |
|------|------|------|
| 传说结局 | 拥有三叉戟或海神甲 | 海神祝福 |
| 生存结局 | 建造≥5建筑且存活≥25天 | 岛的主人 |
| 记忆结局 | 解锁≥5页故事 | 父亲的归来 |
| 普通结局 | 通关30天 | 孤独的生还者 |
| 沉默结局 | 生命归零 | 无人知晓 |

### 5.5 每日小贴士
每天显示一条生存提示:
- "冶炼屋可以自动修复装备!"
- "瞭望塔减少灾害发生率!"
- "防御墙可以减少伤害!"
- "满饱食度完成任务更快!"
- "三叉戟是传说级武器!"
- "收集8页故事可解锁记忆结局!"

## 6. UI/UX 设计

### 6.1 界面层次
```
_draw()
├── menu (主菜单)
├── main (游戏主界面)
│   ├── _draw_bg() — 天空/海洋/昼夜
│   ├── _draw_island() — 多层岛屿渲染
│   ├── 粒子系统 (灾害/特效)
│   ├── _draw_minimap() — 小地图
│   ├── _draw_main_panel() — 角色面板
│   ├── _draw_ui() — 状态栏
│   ├── _draw_action_buttons() — 操作按钮
│   ├── 消息日志
│   └── 悬浮面板 (inventory/craft/build/achievement/quest_log/drift)
├── combat (战斗)
├── game_over
└── victory (结局)
```

### 6.2 视觉效果
- **天空**: 昼夜渐变 + 云朵飘动
- **海洋**: 8层动态波浪
- **星空**: 80颗星星闪烁 + 月亮
- **岛屿**: 沙滩渐变 + 草地 + 礁石 + 树/灌木
- **建筑**: 窗户/门/屋顶细节 + 冶炼屋火焰动画

### 6.3 快捷键
| 按键 | 功能 |
|------|------|
| S | 存档 (Slot1) |
| L | 读档 (Slot1) |
| F | 捕捞 |
| E | 探索 |
| ESC | 关闭面板 / 返回菜单 |
| 任意键 | 跳过开场动画 |

### 6.4 多存档系统
- 3 个存档槽 (save1.json / save2.json / save3.json)
- 菜单显示各槽存档天数
- `_load_slot_list()` 启动时扫描已有存档

## 8. 代码架构

### 8.1 类结构
```
@dataclass Building — 建筑 (name, x, y)
@dataclass Particle — 粒子 (x, y, vx, vy, life, color, size)
@dataclass Enemy — 敌人 (name, max_hp, atk, dfs, exp, loot)
@dataclass Player — 玩家状态 (生命/饱腹/背包/建筑/成就/故事/耐久等20个字段)
class Parts — 粒子系统管理器 (burst/rain/tornado/plume + update/draw)
@dataclass Disaster — 灾害 (type, duration, damage)
class Quest — 任务 (qtype, title, desc, target, target_count, reward)
class StoryPage — 故事页 (id, title, content, mood)
class Game — 主控制器 (所有游戏逻辑)
```

### 8.2 关键数据
- `ITEMS` — 物品元数据 (类型/名称/颜色)
- `BUILDINGS` — 建筑数据 (7种)
- `ENEMIES` — 敌人数据 (5种)
- `ACHIEVEMENTS` — 成就数据 (7种)
- `RECIPES` — 配方数据 (19种)
- `DRIFT_BOTTLES` — 故事数据 (8页)
- `ENDINGS` — 结局数据 (5种)
- `PARTICLE_CONFIGS` — 粒子配置

### 8.3 核心流程
```
run() → _handle_events() → _update(dt) → _draw()
          ↓
      intro_done?
        ↓ no    → _draw_intro_screen() (开场动画)
        ↓ yes
      state machine:
        menu → _draw_menu()
        main → _draw() → _draw_bg/_island/_ui/_buttons + overlays
        combat → _draw_combat_ui()
        game_over / victory → _draw_*_screen()
```
